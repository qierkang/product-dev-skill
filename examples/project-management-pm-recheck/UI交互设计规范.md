# 项目管理 UI 交互设计规范

## 元信息

- 版本：`v1.0`
- 日期：`2026-04-11`
- 编写人：`qierkang+codex`
- 状态：`草稿`

## 1. 全局规范

- 目标用户：项目经理、项目成员、部门负责人
- 业务目标：在同一套 PC Web 页面中完成项目创建、查看、编辑、归档、删除，以及成员、任务、文件的统一维护
- 终端范围：`PC Web 首发；Android App 与微信小程序仅保留适配位，不在本轮落地`
- 页面类型：`列表页 / 详情页 / Tab 页 / 弹窗`

## 2. 页面定位

- 主任务：在项目看板中快速定位项目，在项目详情中完成主表与子表的统一维护
- 高频操作：查询、重置、添加、查看、编辑、保存、提交、添加成员、添加任务、上传附件、下载附件、归档、删除
- 关键风险点：
  - 查看态与编辑态按钮混乱导致误操作
  - 项目阶段与项目状态口径混杂
  - 项目进度展示与任务完成数口径不一致
  - 详情页残留列表页模板文案，影响真实交互判断

## 3. 三端范围矩阵

|-----------|--------|-------------|------------|------|

| 项目详情基础信息 | 本期实现 | 预留 | 预留 | 主表表单区 |
| 任务管理 | 本期实现 | 预留 | 预留 | Tab + 弹窗 |

## 4. 全局展示规范

- 编码与名称展示格式：
  - 项目编号单独展示为只读字段，不与项目名称拼接。
  - 人员展示默认用 `姓名`，需要区分身份时补充 `编码` 或 `部门`。
- 系统字段默认展示格式：
  - 创建时间、更新时间统一展示 `yyyy-MM-dd HH:mm:ss`
  - 项目进度统一展示为 `进度条 + 百分比`
- 列表与详情的显示差异：
  - 列表页展示项目摘要卡片
  - 详情页展示完整主表与 Tab 子表
- 只读字段表现形式：
  - 项目编号、创建人、创建时间、更新时间、项目进度使用只读样式

## 5. 页面结构与操作路径

### 5.1 页面分区

- 头部区：页面标题、返回入口、主操作按钮
- 搜索筛选区：项目名称、部门、日期区间、项目阶段
- 数据展示区：项目卡片看板
- 操作区：添加、查询、重置、编辑、保存、提交、归档、删除
- 明细区：任务管理、项目成员、项目动态、项目文件四个 Tab

### 5.2 核心交互路径

| 场景 | 操作步骤 | 预期结果 |
|------|----------|----------|
| 新建项目 | 点击添加 -> 进入详情新增态 -> 填写主表 -> 维护成员/任务/文件 -> 点击提交 | 返回看板并新增项目卡片 |
| 保存草稿 | 新增态或编辑态 -> 点击保存 | 返回看板并保留草稿状态 |
| 编辑项目 | 点击卡片或编辑 -> 进入详情编辑态 -> 修改后提交 | 更新原项目卡片 |
| 添加成员 | 点击添加成员 -> 选择人员 -> 提交 | 项目成员子表新增记录 |
| 添加任务 | 点击添加任务 -> 填写任务弹窗 -> 提交 | 任务子表新增记录，并同步到负责人任务域 |
| 上传附件 | 点击添加文件或上传附件 -> 选择文件 -> 上传成功 | 项目文件子表新增记录 |
| 下载附件 | 在项目文件区点击下载 | 下载指定附件 |
| 归档项目 | 点击归档项目 -> 校验未完成任务 -> 确认 | 状态更新为已归档并返回看板 |
| 删除项目 | 点击删除项目 -> 弹出确认框 -> 确认 | 状态更新为已删除并返回看板 |

## 7. 弹窗 / 选择器 / 独立页规范

- 哪些字段使用弹窗选择器：
  - 项目负责人
  - 添加成员
  - 新建任务中的负责人如需大数据量选择，可升级为弹窗
- 哪些字段使用独立选择页：
  - 当前 PC 原型不需要独立选择页，移动端适配时可转独立页
- 关闭后是否保留草稿态：
  - 成员弹窗、任务弹窗关闭后保留详情页上下文
  - 详情页关闭前如果有未保存修改，需二次确认
- 是否支持二次改选：
  - 项目负责人支持改选
  - 项目成员支持增删
  - 任务负责人支持重新选择

## 8. 关闭与保存交互规范

- 保存成功：
  - 关闭详情页
  - 返回项目看板
  - 状态写为 `草稿`
- 保存失败：
  - 保持当前页
  - 定位到错误字段并给出明确提示
- 关闭弹窗：
  - 任务弹窗、成员弹窗、上传附件弹层关闭后不丢失详情页已录入内容
- 有未保存修改时：
  - 点击取消、关闭或返回，必须提示“是否放弃本次修改”

## 9. 表单校验与联动

| 字段 | 校验规则 | 联动触发条件 | 联动结果 | 备注 |
|------|---------|-------------|---------|------|
| 项目名称 | 必填，长度 <= 20 | 点击保存/提交 | 校验失败阻断操作 | - |
| 所属部门 | 必须选择有效部门 | 保存/提交前 | 校验失败阻断操作 | - |
| 截止日期 | 不得小于开始日期 | 选择日期后 | 立即提示并阻断提交 | - |
| 项目阶段 | 必须在正式枚举范围内 | 页面加载和提交前 | 默认规划中，查看态只读 | 污染状态值不进入枚举 |
| 添加成员 | 不允许重复成员 | 选择成员后 | 已存在时提示冲突 | - |
| 任务负责人 | 必须来自项目成员 | 打开任务弹窗或成员变更后 | 只展示项目成员可选项 | - |
| 任务优先级 | 必须选择 | 任务提交前 | 校验失败阻断提交 | 四种优先级 |
| 项目进度 | 系统计算展示 | 任务状态变化后 | 刷新进度条和百分比 | 计算公式待确认 |

## 12. 视觉与端差异说明

- PC：
  - 列表页采用卡片式看板，便于展示项目进度和成员摘要
  - 详情页采用上方主表、下方 Tab 的管理台结构
  - 归档、删除采用危险样式按钮
- Android：
  - 后续建议拆成基础信息卡片 + 分段列表
- 小程序：
  - 后续建议采用 Cell 布局和轻量弹层
- 危险操作与主操作的视觉层级：
  - `提交` 为主按钮
  - `保存` 为次按钮
  - `归档项目` 与 `删除项目` 为危险按钮

## 13. 特殊说明

- 是否依赖内部公共接口回填：`是，部门数据、人员列表、项目编号、任务同步、文件上传均依赖内部能力`
- 是否存在系统字段只读但必须展示：`是，项目编号、创建人、创建时间、更新时间、项目进度`
- 是否存在审批状态 / 编码 / 字典值特殊展示：`是，项目阶段使用正式枚举；项目状态按系统过程值处理；项目编号和任务编码建议系统生成`
- 原型中的查询条件残留、非项目域状态值和文案笔误均不直接进入正式 UI 口径。

## 14. 验收要点

- 查看态下主表与子表字段不可编辑。
- 点击保存后必须返回列表页并保留草稿状态。
- 点击提交时，新增态新增项目，编辑态回写原项目。
- 添加成员后项目成员子表必须新增记录，且不能重复。
- 添加任务后任务子表必须新增记录，并同步到负责人任务域。
- 上传附件后项目文件子表必须新增记录，且支持下载。
- 归档项目时若存在未完成任务，必须阻断并提示原因。
- 删除项目时必须弹出“删除后不可恢复”的确认提示。
