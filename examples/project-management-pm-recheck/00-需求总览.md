- 建立项目从创建、保存草稿、提交、查看、编辑到归档、删除的完整维护闭环。
- 在一个项目详情页内统一维护项目基础信息、项目成员、任务管理、项目文件和项目动态，减少跨页面跳转。
- 明确原型中的污染项、文案笔误和待确认项，保证需求文档不把模板噪音直接带入正式需求。

## 5. 当前结构判断

- 页面范围：`项目看板 + 项目详情 + 新建任务弹窗 + 添加成员弹窗 + 项目文件上传下载`
- 终端范围：`当前原型以 PC Web 为主，Android App / 微信小程序仅预留，不在本轮落地`
- 数据结构：`主表 + 多子表`
- 是否涉及主表 / 子表 / 孙表：`是，主表为项目；子表至少包含项目成员、任务管理、项目文件；项目动态当前按只读子表处理`
- 是否涉及系统字段自动带出：`是，项目编号、创建人、创建时间、更新人、更新时间、项目进度、任务编码建议系统生成`
- 是否存在文案污染或模板残留：`是，列表页出现疑似非项目域状态值；详情页出现列表页查询条件残留；任务按钮说明存在明显笔误；原型存在“人员编码编码”重复词`

## 6. 待确认项

- `项目状态` 与 `项目阶段` 是否为两套独立字段。当前文档按“项目阶段”为展示与筛选主口径，`待审批 / 待付款 / 在借 / 逾期 / 已归还` 视为模板污染。
- `项目进度` 的计算口径未在原型中写清。当前按系统计算展示字段处理，后续需明确是按任务完成率、里程碑完成率还是人工录入。
- `任务执行人` 是否与 `任务负责人` 同字段。原型文案不够完整，当前文档按“负责人必填、执行人按需扩展”处理。
- `项目动态` 仅展示了区块，没有完整字段结构。当前按只读日志区处理，后续如需落库需单独补充字段设计。
