# 模板索引与产物规范

本目录维护 `product-dev-skill` 的标准文档模板。

模板来源以 `picasso-dev-skill/shared/templates` 为主，尤其是 `需求文档模板.md`，保留它的字段来源、系统字段、主子表层级和按钮逻辑等强约束结构。

UI 设计部分优先协同外部 `ui-ux-pro-max`；若当前运行端不可识别该 skill，则回退到 `shared/references/design/` 的内置规则。

## 当前模板清单

| 模板 | 作用 | 说明 |
|------|------|------|
| `需求总览模板.md` | request 导航页 | 汇总输入资产、页面清单、待确认项 |
| `需求文档模板.md` | 主需求文档模板 | 当前仓库最核心模板 |
| `UI交互设计模板.md` | UI 交互文档模板 | 基于需求文档继续展开 |
| `manifest模板.json` | request 元信息模板 | 机器可读入口 |
| `stage-status模板.json` | 阶段状态模板 | 记录当前 Gate 状态 |
| `需求文档补充规则.md` | 需求文档附加约束 | 与主模板配套使用 |

## 写作要求

1. 每份文档必须保留：版本、日期、编写人、状态。
2. 需求文档优先写清：字段来源、展示格式、编辑规则、联动逻辑、状态变化。
3. UI 文档优先写清：页面分区、操作路径、按钮显隐、表单校验、空态与异常态。
4. UI 文档现在还必须补：风格方向、组件策略、动效与反馈、可访问性与响应式要求。
4. 模板里已经存在的系统字段、层级字段、弹窗选择器说明，不要在产出时省掉。
