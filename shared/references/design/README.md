# 设计参考索引

本目录是 `product-dev-skill` 的内置设计兜底层。

如果当前运行端已经安装并可识别 `ui-ux-pro-max`，优先先用外部 design skill 给出：

1. 风格方向
2. 色板 / 字体
3. token 基线
4. 组件策略
5. 动效与反馈
6. 可访问性要求

如果当前运行端没有该外部 skill，则按以下文件完成同一件事：

1. `风格方向与信息层级.md`
2. `多端适配基线.md`
3. `动效与反馈规则.md`
4. `平台差异速查.md`

这里的目标不是替代外部高质量设计 skill，而是保证 `product-dev-skill` 在任何机器上都不会因为“缺少外部设计能力”而退化成只写字段的文档机。

即使当前运行端还能识别 `frontend-design` 等通用设计 skill，也不得把它们当成 `ui-ux-pro-max` 缺失时的默认兜底。
