# 风格方向与信息层级

## 目标

把原型页面从“结构描述”提升到“设计意图表达”。

## 最低要求

UI 文档至少要明确：

- 页面气质关键词
- 风格方向
- 主信息区与次信息区
- 主操作和危险操作的优先级

## 默认风格方向

### 1. 高密度业务台

适用：

- PC 管理页
- 列表页
- 详情页

重点：

- 扫描效率
- 信息分组
- 查询和批量处理效率

### 2. 流程型表单页

适用：

- 新增 / 编辑
- 主子表录入
- 分步骤页面

重点：

- 一屏只讲一类决策
- 保存 / 提交 / 返回分层

### 3. 结果展示页

适用：

- 报告页
- 结果页
- 活动结果页

重点：

- 首屏结论
- 情绪价值
- 分享 / 下载 / 下一步

## 信息层级规则

1. 先结论，后说明
2. 先主任务，后扩展信息
3. 同屏不并列多个主 CTA
4. 重要状态必须独立呈现，不埋在正文里
