# UI Gate

## 必备文件

- `需求文档.md`
- `UI交互设计规范.md`
- `stage-status.json`

## 最低检查点

- UI 文档必须包含：`设计策略与风格方向`
- UI 文档必须包含：`设计 Token 与组件策略`
- UI 文档必须包含：`页面结构与操作路径`
- UI 文档必须包含：`状态反馈与动效规范`
- UI 文档必须包含：`表单校验与联动`
- UI 文档必须包含：`状态与按钮显隐`
- UI 文档必须包含：`可访问性与响应式要求`
- UI 文档必须包含：`验收要点`

## 通过标准

- UI 文档与需求文档无明显冲突
- 按钮、状态、只读态、弹窗说明齐备
- 已明确风格方向、组件策略和多端差异
- 已具备进入设计评审或开发交底的基础
