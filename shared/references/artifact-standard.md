# 交付物标准

## 最低交付清单

1. `00-需求总览.md`
2. `需求文档.md`
3. `UI交互设计规范.md`
4. `manifest.json`
5. `stage-status.json`

## 建议辅助产物

1. `原型解析摘要.md`
2. `assets/` 原型输入资产索引
3. `logs/` 过程记录

## 文档元信息硬要求

- 每份 Markdown 文档必须有：`版本`、`日期`、`编写人`、`状态`
- 默认作者可通过 `.env` 或 `profile.yaml` 配置
- 需求总览必须写清：输入资产、页面清单、待确认项
- 需求文档必须写清：字段来源、按钮逻辑、表单定义、业务规则
- UI 文档必须写清：页面结构、状态规则、校验联动、验收要点
